# PART 1 — SETTLEMENT: NO PHANTOM WINS  (most urgent)
################################################################################
ROOT CAUSE (dashboard/settle.py): `_settle_leg` returns None when a player's
actual stat isn't found and the docstring calls it "void". `settle_bets` then does
`elif all(lr["hit"] for lr in non_void): status="won"` — i.e. it settles the multi
on only the gradeable legs. So "Carlton to win (hit) + 2 prop legs (no data ->
void)" settled WON at full odds. Missing data is being conflated with did-not-play.

1A. A multi auto-settles ONLY when EVERY leg is definite:
    - WON  = every leg HIT.
    - LOST = at least one leg a definite MISS (rest known).
    - Any leg UNGRADEABLE (no data, not a confirmed DNP) -> the WHOLE bet stays
      PENDING with a flag naming the ungradeable leg(s); NOT counted in P&L.
    REMOVE the "settle on the non-void subset" path. Never ignore an unresolved leg.
1B. Distinguish did-not-play from no-data:
    - GENUINE VOID (allowed): round player stats ARE populated AND the player was
      named in the lineup but has no row -> did-not-play (prefer flagging for a
      quick manual confirm).
    - UNGRADEABLE -> PENDING: round stats empty (not published yet) OR player not
      found in a populated set (likely name mismatch); H2H/total with no result.
      Log leg name + reason.
1C. Re-grade existing mis-settled bets: any bet currently "won"/"void" that still
    contains an ungradeable leg reverts to PENDING (clear payout/profit/settled_at)
    so the current phantom +$11 disappears and it re-settles correctly when data
    arrives.

ROOT CAUSE (build/report.py): the cap only counts marks legs with NO book price
(`odds_book.get(leg.name) is None`). Once Sportsbet/TAB prices a marks market the
leg bypasses the cap; the Sportsbet ladder is all-priced so the cap never applies
-> multis with 2-3 marks legs.

2A. Count EVERY marks leg toward the cap (remove the price condition; rename
    `_n_model_marks` -> `_n_marks`). Apply `_n_marks <= MAX_MARKS_LEGS_PER_MULTI`
    in BOTH search_match_sgms (model ladder) AND search_market_sgms (Sportsbet
    ladder). Keep MAX_MARKS_LEGS_PER_MULTI = 1 (Ben can set 0). Keep STAT_PREFERENCE
    (disposals 1.0, goals 0.85, tackles 0.7, marks 0.35) as the secondary sort —
    "stay to disposals/goals/tackles", marks last. If a band can't fill with <=1
    marks + distinct players, take the closest qualifying combo (still <=1 marks);
    note a skipped band only if genuinely unfillable — never exceed the cap.
2B. Show the computed greasiness value per game on the DASHBOARD (and keep the
    report header tag): e.g. "greasiness 0.00 (dry)" / "0.55 (greasy)" / "0.80
    (wet)" — so Ben can see whether a slippery game (e.g. Collingwood) was flagged.
2C. Per-game greasiness override (forecast under-calls dewy nights): a small JSON

2D. Make DEFAULT_RAIN_MULTIPLIERS config-tunable; consider marks ~0.85 -> ~0.78 at
