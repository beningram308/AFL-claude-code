--------------------------------------------------------------------------------
STEP 1 — Put promo Total EV on every ladder rung (count from sim masks, not the
         independence formula)
--------------------------------------------------------------------------------
In `build/report.py search_match_sgms`, each selected rung's legs already carry
aligned per-iteration sim masks (`LegCandidate.mask`); the rung is same-game, so
the masks are jointly valid and capture the real correlation.

For each rung compute, directly from the masks (NOT analytic p1*p2*p3):
  - stack the legs' boolean masks -> per-iteration count of WINNING legs
  - P(all win)        = mean(n_win == n_legs)
  - P(exactly 1 loss) = mean(n_win == n_legs - 1)
  - P(2+ loss)        = 1 - P(all win) - P(exactly 1 loss)
Then:
  - Promo EV  = P(exactly 1 loss) * R           (R = refund cash-value factor)
  - Total EV  = base EV + Promo EV, where base EV uses the rung's existing
                joint_prob and book/fair odds (per $1 staked).
  - R comes from config `BONUS_BET_FACTOR` (keep it tunable; default ~0.70). Add a
    note/field so a realised R can be recorded later.
Add these to each rung's record: p_all_win, p_one_loss, p_two_plus_loss, promo_ev,
total_ev. Gate the promo term on a configurable minimum-legs rule (default 3); the
ladder is 3-leg so all rungs qualify.

Keep the band ladder structure exactly as is (do NOT re-select rungs to maximise
the refund — that's the wrong objective). Promo is an added metric. The **VALUE
PICK** tag should now rank by **Total EV edge** (book) rather than base edge, when
real odds exist.

--------------------------------------------------------------------------------
STEP 2 — Multi-outcome Kelly staking (replace binary Kelly for promo multis only)
--------------------------------------------------------------------------------
In `build/staking.py`, add:
    def multi_outcome_kelly(p_win, p_one_loss, p_dead, multi_odds, refund_factor,
                            fraction=KELLY_FRACTION, cap=KELLY_PER_BET_CAP) -> float
maximising expected log-growth across the three outcomes:
    g(f) = p_win*ln(1 + f*(M-1)) + p_one_loss*ln(1 + f*(R-1)) + p_dead*ln(1 - f)
    net returns:  win -> (M-1),  one loss -> (R-1),  2+ loss -> -1   (M=multi_odds, R=refund_factor)
Solve g'(f)=0 numerically (scipy.optimize.brentq / minimize_scalar on -g), clamp
f to [0, cap], then apply the fractional multiplier and existing caps. Note R<1 so
the one-loss branch is a partial loss, not a refund of full stake.
Keep binary `kelly_fraction` for ordinary win/lose singles. Use
`multi_outcome_kelly` for promo-eligible multis. Surface a **suggested stake** per
ladder rung (fraction of bankroll) using it.

Acceptance for this step: for a rung that is negative on a standalone basis but
+Total-EV with the promo, binary Kelly returns ~0/negative while multi_outcome
Kelly returns a positive, sensibly-capped stake.
