strongest promo multis, quarter-Kelly wants MORE than 3u, so they clip to 3u and
all look identical. Question: is 3u the right ceiling, or would 1.5u / 2u / 4u

two ways — a realized replay and a probabilistic sim — instead of opinion.

This is a WHAT-IF sweep. It must not change how the live bot prices, sizes, or
settles anything. It only re-reads past rounds and re-scores them at different

VERSION A — REALIZED REPLAY  (what LITERALLY would have happened)
------------------------------------------------------------------------
For each candidate cap in the sweep:
1. Re-size every rung by calling size_multi_units() with unit_max = candidate
   (everything else identical: KELLY_FRACTION, promo_refund_cap, unit_step,
   longshot cap, PROMO_EV_MIN eligibility floor). Keep the round-cap OFF to match
   the current live setting, but expose it as a parameter (see sweep).
2. Settle each rung against ACTUAL results by grading its legs from the box-score
   stats (reuse existing grader). Apply the real promo/stake-back settlement:
     - all legs hit            -> profit = stake * (book_odds - 1)
     - exactly one leg misses  -> stake-back refund; net = -(1 - refund_factor)*stake
                                  (refund_factor = BONUS_BET_FACTOR, ~0.75)
     - two or more miss        -> net = -stake
   Straight (non-promo) rungs settle plain win/lose, no refund. Use the tag/branch
   data already on the rung to decide which settlement applies — same as the live
   ledger.
3. Chain rounds in date order into one bankroll curve (start = live BANKROLL 1500).
4. Report per cap: n bets, total staked, total returned, net profit, ROI %,
   ending bankroll, and max drawdown (peak-to-trough on the curve).

Honesty note to print in the report: this is ~5 rounds, so it's a SMALL, luck-heavy
sample — one or two multis dominate. Treat Version A as "what happened," not "what's
optimal."

------------------------------------------------------------------------
VERSION B — PROBABILISTIC BANKROLL SIM  (is the EDGE real, luck aside)
------------------------------------------------------------------------
For each candidate cap:
1. Take the same set of recommended rungs, sized at that cap.
2. Monte Carlo N = 10,000 bankroll paths. Each path replays the bets in date
   order; for each bet draw its outcome from its OWN modelled branch probs:
     win  with p_all_win        -> +stake*(book_odds-1)
     refund with p_one_loss     -> -(1 - refund_factor)*stake
     dead with p_dead           -> -stake
   (straight bets: win with joint_prob, else lose.)
3. Two staking modes, report both:
     (a) FIXED unit ($15) off the live 1500 bankroll — matches current behaviour.
     (b) COMPOUNDING — recompute stake off the running bankroll each bet (truer
         Kelly growth read).
4. Aggregate across the 10k paths per cap: median ending bankroll, mean, 5th &
   95th percentile, median max drawdown, P(ending below start), P(drawdown > 20%).

This leans on the model's probabilities rather than which bets happened to land,
so it reflects the underlying edge — PROVIDED the model is calibrated.
