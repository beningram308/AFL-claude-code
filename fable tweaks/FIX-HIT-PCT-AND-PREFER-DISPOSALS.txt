--------------------------------------------------------------------------------
PART B — Prefer disposals, minimise marks (configurable)
--------------------------------------------------------------------------------
Ben prefers disposals legs and wants fewer marks legs in the multis. Make this a
soft, tunable preference — not a hard ban — applied in the SHARED combo builder
so BOTH ladders (Model + Sportsbet) inherit it.

B1. config.py — add:
      # Higher = more preferred when the selector has a choice between combos.
      STAT_PREFERENCE = {"disposals": 1.0, "goals": 0.85, "tackles": 0.7, "marks": 0.35}
      # Hard cap on model-only marks legs in a single multi (priced marks legs,
      # i.e. ones with a real book price, are exempt and always allowed).
      MAX_MARKS_LEGS_PER_MULTI = 1     # set 0 to remove model-only marks entirely

B2. In build/report.py search_match_sgms combo handling, AFTER the existing
    band-landing rule and the distinct-subject filter:
      - DROP any combo whose count of model-only marks legs exceeds
        MAX_MARKS_LEGS_PER_MULTI (a marks leg with a real book_odds is exempt).
      - Among the combos that still satisfy band + distinct-subject + marks-cap,
        rank by a PREFERENCE SCORE = sum of STAT_PREFERENCE[leg.market_stat] across
        the combo's legs (disposals win, marks lose), used as a SECONDARY key —
        the primary "land at/above the band, closest from above" rule is unchanged,
        and the stable sorted-leg-names tie-break stays last for determinism.
    Net effect: when two combos both fill a band, the one with more disposals /
    fewer marks is chosen; marks only appear when needed to fill a band or when
    they carry a real price.
B3. Do NOT change which lines/legs are *eligible* (the menu, the 0.78 cap, the
    distinct-player rule all stay). This only changes PREFERENCE among valid
    combos, so the ladder still fills every band.
